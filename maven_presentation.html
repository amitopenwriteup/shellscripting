maven clean

What it does:

Removes all files generated during the previous build from the target directory.

mvn clean

When to use:

  • Before starting a fresh build
  • When you want to ensure no old artifacts remain
  • To troubleshoot build issues caused by stale files
  • Before deploying to production

What gets deleted:

  • Compiled .class files
  • JAR/WAR files
  • Test reports
  • Generated resources

maven package

What it does:

Compiles your code, runs tests, and packages the compiled code into a distributable format (JAR, WAR, EAR).

mvn package

Build Lifecycle Phases Executed:

validate compile test package

Output:

  • Creates a JAR file (for Java applications)
  • Creates a WAR file (for web applications)
  • Located in the target/ directory
  • Named as: artifactId-version.jar
4 / 10

maven test

What it does:

Runs unit tests using the testing framework (JUnit, TestNG) configured in your project.

mvn test

Key Features:

  • Executes all test classes in src/test/java
  • Does not package the application
  • Generates test reports in target/surefire-reports
  • Fails the build if any test fails

Skip Tests:

mvn package -DskipTests

Compiles tests but doesn't run them

mvn package -Dmaven.test.skip=true

Skips both compilation and execution of tests

maven validate

What it does:

Validates that the project structure is correct and all necessary information is available.

mvn validate

Validation Checks:

  • POM.xml syntax is correct
  • Required project information is present
  • Project dependencies are resolvable
  • Plugin configurations are valid
  • Project structure follows Maven conventions

When to use:

  • Before committing changes to version control
  • To quickly check project configuration
  • As part of CI/CD pipeline initial checks
  • After modifying POM.xml

POM.xml - Project Object Model

What is POM.xml?

The Project Object Model (POM) is the fundamental unit of work in Maven. It's an XML file that contains information about the project and configuration details used by Maven to build the project.

Key Purposes:

  • Defines project structure and metadata
  • Manages project dependencies
  • Configures build plugins
  • Specifies project repositories
  • Defines build profiles
<project xmlns="http://maven.apache.org/POM/4.0.0"> <!-- POM content goes here --> </project>

POM.xml - Basic Elements

<project> <modelVersion>4.0.0</modelVersion> <groupId>com.example</groupId> <artifactId>my-app</artifactId> <version>1.0-SNAPSHOT</version> <packaging>jar</packaging> <name>My Application</name> <description>A sample application</description> </project>

Element Descriptions:

  • modelVersion: POM model version (always 4.0.0)
  • groupId: Organization/group identifier
  • artifactId: Project name/identifier
  • version: Project version
  • packaging: Package type (jar, war, ear, pom)

POM.xml - Dependencies Section

Managing Project Dependencies:

<dependencies> <dependency> <groupId>org.springframework.boot</groupId> <artifactId>spring-boot-starter-web</artifactId> <version>3.2.0</version> </dependency> <dependency> <groupId>junit</groupId> <artifactId>junit</artifactId> <version>4.13.2</version> <scope>test</scope> </dependency> </dependencies>

Dependency Scopes:

  • compile: Default, available in all classpaths
  • test: Only for testing (JUnit, Mockito)
  • provided: Expected by JDK or container
  • runtime: Not needed for compilation

POM.xml - Build & Plugins

<build> <plugins> <plugin> <groupId>org.apache.maven.plugins</groupId> <artifactId>maven-compiler-plugin</artifactId> <version>3.11.0</version> <configuration> <source>17</source> <target>17</target> </configuration> </plugin> <plugin> <groupId>org.apache.maven.plugins</groupId> <artifactId>maven-surefire-plugin</artifactId> <version>3.0.0</version> </plugin> </plugins> </build>

Common Plugins:

  • maven-compiler-plugin: Compiles Java source
  • maven-surefire-plugin: Runs unit tests
  • maven-jar-plugin: Creates JAR files

Complete POM.xml Structure

<project> <modelVersion>4.0.0</modelVersion> <!-- Project Coordinates --> <groupId>com.example</groupId> <artifactId>my-app</artifactId> <version>1.0.0</version> <packaging>jar</packaging> <!-- Project Information --> <name>My Application</name> <description>Sample Maven Project</description> <!-- Properties --> <properties> <maven.compiler.source>17</maven.compiler.source> <maven.compiler.target>17</maven.compiler.target> <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding> </properties> <!-- Dependencies --> <dependencies>...</dependencies> <!-- Build Configuration --> <build> <plugins>...</plugins> </build> </project>
